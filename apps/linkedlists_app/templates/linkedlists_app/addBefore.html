[Add Node Before]

Input: Value of new node, the value of the node to add your new node before

Output: The Sll instance

Since the data structure of Slls do not allow for the indexing of nodes, you must find a way to identify the specific location where a node needs to be added. You will need to implement a runner to iterate through the elements of the Sll to reach the point where you want to add your Sll. You will additionally need a checker that iterates one spot ahead of the runner to confirm or deny whether the position of the checker is where the node needs to be added. Remember, each node in a Singly Linked Lists only had one pointer: top the next node. Once you've iterated past a node, there's no way to go back to it, so in order to add a node before a specific point in the Sll, you have to have a runner and a checker to keep track of where you are.

Assumptions:

The user will submit the correct input values and types.

Edge Cases:

What do you do when you are given an empty Sll?

What happens if the chk value (the value to insert the node before) doesn't exist?

Sll.prototype.addBefore = function(val, chk) {
	var addNode = new Node(val);
	if(!this.head) {
		return false;
	} else {
		var runner = this.head;
		while (runner.next) {
			if (runner.next.val == chk) {
				addNode.next = runner.next;
				runner.next = addNode;
				return self;
			}
		}
		return false;
	}
}
